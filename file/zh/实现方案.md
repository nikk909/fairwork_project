## 4. 共用：货币换算（`currency_utils.py`）

### 4.1 输入列

`budget_min`、`budget_max`、`bid_avg`、`currency_exchange_rate`、`currency_code`

### 4.2 公式

`currency_exchange_rate` 相对 USD。先换到 USD 再换到 EUR，等价于：

```text
金额_EUR = 金额_原币 × rate_原币 / rate_EUR
```

其中 `rate_原币` = 该行的 `currency_exchange_rate`，`rate_EUR` = 欧元行的汇率。

### 4.3 算法步骤

```python
def add_eur_columns(df: pd.DataFrame) -> pd.DataFrame:
    out = df.copy()
    eur_rates = out.loc[out["currency_code"] == "EUR", "currency_exchange_rate"].dropna()
    # 取众数；若众数不可用则取首个非空
    rate_eur = eur_rates.mode().iloc[0]
    rate_row = out["currency_exchange_rate"]
    for col in ("budget_min", "budget_max", "bid_avg"):
        out[f"{col}_eur"] = out[col] * rate_row / rate_eur
        # 原值为 NaN → 对应 *_eur 保持 NaN（pandas 乘法自然传播）
    return out
```

### 4.4 输出列

| 新列 | 含义 |
|------|------|
| `budget_min_eur` | 预算下限（欧元） |
| `budget_max_eur` | 预算上限（欧元） |
| `bid_avg_eur` | 平均投标额（欧元） |

### 4.5 硬性约定

- 后续 Fair Pay / 议价 **禁止**再用未换算的 `budget_*` / `bid_avg` 算金额
- `rate_EUR` 必须能解析；若表中无 EUR 行，应 `raise ValueError` 并提示检查数据

## 6. 议题 2：`language_scoring.py`

### 6.1 编码定义

| 字段 | 取值 | 含义 |
|------|------|------|
| `de_level` | 0–3 | 0=不需要；1=A1/A2；2=B1/B2；3=C1/C2 |
| `en_level` | 1–3 | 1=A1/A2；2=B1/B2；3=C1/C2（**无 0**：默认至少入门） |
| `de_reason` / `en_reason` | string | 判定短理由（须中英双语；AI 结果前缀 `[AI辅助]`） |

对**全部岗位**写出以上四列。

### 6.2 判定流水线（严格优先级）

对每一行，扫描文本来源：`job_names`、`title`、`description`（可拼成一个大写无关的搜索串，但 reason 里注明命中出处）。

```text
1. 规则层（CEFR 正则 + 技能标签 + 词汇库）
      → 同语言多命中：取 min(水平)
      → 理由前缀 [rule]，格式「中文 / English」
2. 若 de_level 与 en_level 均已确定 → 结束
3. 否则（含「未发现德语要求」「默认英语入门」等软默认情形）→ 调用 AI/DeepSeek
      → 填缺失项；理由前缀 [AI辅助]；理由须中英双语
4. 发帖语言兜底（在 AI / 规则之后强制执行）：
      language == "en" → en_level = max(en_level, 1)
      language == "de" → de_level = max(de_level, 1)
5. 若仍缺项（无 Key / API 失败）→ fallback：
      缺 de → de_level=0，de_reason="[fallback] 中文 / English"
      缺 en → en_level=1，en_reason="[fallback] 中文 / English"
      并 logging.warning
```

**禁止**：规则层单独写出 `[rule] no German requirement found` 或 `[rule] default English beginner` 并结束——这两类必须走 **AI 辅助**。

「完整」定义：`de_level` 与 `en_level` 均非空（已赋合法整数）。

### 6.3 规则层细节

#### （A）CEFR 正则（示例）

在命中「德语/英语」上下文附近，或全局匹配后按语言关键词归属：

- 模式示例：`(?i)\b(A1|A2|B1|B2|C1|C2)\b`
- 映射：A1/A2→1，B1/B2→2，C1/C2→3
- 若 CEFR 旁出现 `German` / `Deutsch` / `德语` → 计入 `de`；`English` / `Englisch` / `英语` → 计入 `en`
- 无法判断归属语言时：忽略该 CEFR 命中（交给后续规则或 DeepSeek），避免误标

#### （B）技能标签

`job_names` 中（大小写不敏感、可做简单归一化）：

| 标签示例 | 语言 | 默认水平（无更细修饰时） |
|----------|------|--------------------------|
| German / Deutsch | de | 2（流畅）；若同条另有 CEFR/词汇则再取 min |
| English / Englisch | en | 2 |

仅出现语言名、无程度词时用上表默认；若同字段还有程度词，以词汇库为准并取 min。

#### （C）词汇库（脚本内 dict，键小写）

至少覆盖下列示例（可扩展同义写法）：

| phrase（小写） | 语言 | level |
|----------------|------|-------|
| `no german` / `german not required` / `nicht erforderlich`（需结合 german 上下文） | de | 0 |
| `basic german` / `einfaches deutsch` / `a1`/`a2` + german | de | 1 |
| `fluent german` / `gutes deutsch` / `b1`/`b2` + german | de | 2 |
| `native german` / `muttersprachler` / `c1`/`c2` + german / `verhandlungssicher` | de | 3 |
| `basic english` / `einfaches englisch` | en | 1 |
| `fluent english` / `gutes englisch` | en | 2 |
| `native english` / `muttersprachliches englisch` / `business english`（偏高则 2 或 3，建议 2，不确定取低） | en | 2 或 3 |

匹配：对 `(title + " " + job_names + " " + description).lower()` 做子串查找；同一语言多个命中 → **`min(levels)`**。

## 7. 议题 3：`bargain_analysis.py`

定义与公式完全按 `file/zh/议价能力.md`；金额一律用欧元列。

### 7.1 样本清洗

去掉满足任一条件的行：

- `bid_count` / `bid_avg_eur` / `budget_min_eur` / `budget_max_eur` 缺失
- `budget_max_eur <= budget_min_eur`

输入表须已含议题 2 的 `de_level` / `en_level`（由 `main` 传入 `lang_df`）。

### 7.2 指标与合成

\[
P_i = \frac{bid\_avg\_eur_i - budget\_min\_eur_i}{budget\_max\_eur_i - budget\_min\_eur_i},
\quad
C_i = \log(1 + bid\_count_i)
\]

在**清洗后全样本**上 z-score（`ddof=0`）：

\[
z(P_i)=\frac{P_i-\bar P}{s_P},\quad
z(C_i)=\frac{C_i-\bar C}{s_C}
\]

\[
bargain\_score_i = 0.5 \cdot z(P_i) - 0.5 \cdot z(C_i)
\]

（即 \(\alpha = 0.5\)。）

### 7.3 离散档位（必须做）

按 `bargain_score` 分位数：

| `bargain_tier` | 规则 |
|----------------|------|
| `low` | ≤ Q1 |
| `mid` | Q1–Q3（不含已划入 low/high 的边界按 `pandas.qcut` 或显式分位实现，边界归属保持一致即可） |
| `high` | ≥ Q3 |

便于与语言水平交叉。

### 7.4 fixed / hourly 说明（结论中必须写）

- 本实现：**fixed 与 hourly 合表**做一次 z-score（与议价文档示意一致）
- 局限：`bid_avg` / 预算在两类下含义不同（总价 vs 时薪）；\(P\) 为区间相对位置仍可算，但解释时建议在 `结论.md` 中按 `type` 分组报告均值，或注明「未分类型标准化」

### 7.5 实现示意

```python
import numpy as np

sub = df.dropna(subset=["bid_count", "bid_avg_eur", "budget_min_eur", "budget_max_eur"]).copy()
sub = sub[sub["budget_max_eur"] > sub["budget_min_eur"]]

sub["P"] = (sub["bid_avg_eur"] - sub["budget_min_eur"]) / (
    sub["budget_max_eur"] - sub["budget_min_eur"]
)
sub["C"] = np.log1p(sub["bid_count"])
sub["z_P"] = (sub["P"] - sub["P"].mean()) / sub["P"].std(ddof=0)
sub["z_C"] = (sub["C"] - sub["C"].mean()) / sub["C"].std(ddof=0)
alpha = 0.5
sub["bargain_score"] = alpha * sub["z_P"] - (1 - alpha) * sub["z_C"]
# bargain_tier: 按 Q1/Q3 切 low/mid/high
```
