# 实现方案（简版）

依据：`file/数据处理.md`、`file/议价能力.md`、`deepseek_ai_prompt.md`、`file/fair pay对应打分.md`。

目标：从 `dataset/dataset.csv` 跑通三个议题，结果分别写入 `results/01_fair_pay/`、`results/02_language_vs_score/`、`results/03_language_vs_bargain/`。

---

## 1. 整体流程

```
dataset/dataset.csv
        │
        ▼
   [共用] 读入 + 金额换算 EUR
        │
        ├──────────────────┬──────────────────┐
        ▼                  ▼                  ▼
 fair_pay_analysis   language_scoring   bargain_analysis
   (议题1)              (议题2前置)          (议题3)
        │                  │                  │
        │                  ▼                  │
        │         语言×评分交叉分析            │
        │         (议题2，依赖议题1得分)       │
        │                  │                  │
        ▼                  ▼                  ▼
 results/01_…        results/02_…        results/03_…
```

`main.py` 按顺序调用即可，不引入包、类继承或配置框架。

建议执行顺序：

1. 货币换算（一次，传给后续）
2. Fair Pay（议题1）
3. 语言编码（全样本）→ 语言 × 评分（议题2）
4. 议价 + 语言 × 议价（议题3）

## 3. 共用：货币换算

**输入列：** `budget_min`、`budget_max`、`bid_avg`、`currency_exchange_rate`、`currency_code`

**公式：**

```text
金额_EUR = 金额_原币 × rate_原币 / rate_EUR
```

实现要点：

- 从全表取 `currency_code == "EUR"` 的 `currency_exchange_rate` 作为 `rate_EUR`（若多行取值应一致，取众数或首个非空即可）
- 写出 `budget_min_eur`、`budget_max_eur`、`bid_avg_eur`
- 后续 Fair Pay / 议价**只使用 `*_eur` 列**

可放在 `currency_utils.py` 的一个函数 `add_eur_columns(df) -> df`，由 `main` 先调用再分发。

## 4. 议题 1：`fair_pay_analysis.py`

| 项 | 约定 |
|----|------|
| 样本 | `type == "hourly"`；`budget_min_eur` / `budget_max_eur` 任一缺失则剔除 |
| 衍生 | `hourly_min_eur`、`hourly_avg_eur=(min+max)/2`、`hourly_max_eur` |
| 得分 | `fair_pay_score = 1 if hourly_avg_eur >= 13.9 else 0` |
| 不做 | fixed 岗位打分 |

**输出（`results/01_fair_pay/`）：**

- `fair_pay_hourly.csv`：只保留分析用到的列（id、title、类别、时薪三列、得分等）
- 图（必要）：
  - `score_distribution.png` — 得分 0/1 计数
  - `pass_rate_by_category.png` — 按 `job_category_names` 通过率（样本少的类可合并为 Other）
  - `hourly_avg_boxplot.png` — 平均时薪箱线图（可按得分分组）
- `结论.md`：样本数、通过率、按类别简述

函数建议：`run(df_with_eur) -> fair_pay_df`（返回带得分的 hourly 表，供议题2使用）。

## 5. 议题 2：`language_scoring.py`

### 5.1 全样本编码

对**全部岗位**写出：`de_level`、`en_level`、`de_reason`、`en_reason`。

**判定顺序：**

1. 规则：`job_names` / `title` / `description` 中的 CEFR、技能标签、词汇库 → 多水平取**最低**
2. 仍不完整 → 调 DeepSeek（见 `deepseek_ai_prompt.md`）+ 发帖语言兜底：
   - `language=en` → `en_level = max(en_level, 1)`
   - `language=de` → `de_level = max(de_level, 1)`
3. DeepSeek 理由加 `[deepseek]` 前缀；API Key 读环境变量 / `.env`

词汇库：脚本内一个 dict 即可（如 `fluent german`→2，`native german`→3，`basic english`→1 …），覆盖常见写法；未命中再交给 DeepSeek。

英语无 0（默认至少 1）；德语可为 0。

**中间表：** `results/02_language_vs_score/language_all.csv`（全样本语言列 + 关键字段）。

### 5.2 语言 × Fair Pay 评分

| 项 | 约定 |
|----|------|
| 样本 | 议题1 已打分的 hourly（与 `fair_pay_df` 按 `id` 合并） |
| 分析 | 交叉表：de/en 各 level 的样本数、得分均值/通过率 |
| 图 | 如 `de_level_vs_pass_rate.png`、`en_level_vs_pass_rate.png` |
| 文 | `结论.md` |

函数建议：`score_languages(df) -> df`，`analyze_vs_fair_pay(lang_df, fair_pay_df)`。

## 6. 议题 3：`bargain_analysis.py`

按 `file/议价能力.md`：

1. 用欧元列；剔除 `bid_count` / `bid_avg_eur` / `budget_min_eur` / `budget_max_eur` 缺失，或 `budget_max_eur <= budget_min_eur`
2. \(P = (bid\_avg - min) / (max - min)\)，\(C = \log(1 + bid\_count)\)
3. 全清洗样本上 z-score，`bargain_score = 0.5 * z_P - 0.5 * z_C`
4. 可选：按四分位切高/中/低

与语言合并后做交叉（德语/英语 level × bargain 均值或分组分布）。

**输出（`results/03_language_vs_bargain/`）：**

- `bargain_scored.csv`
- 图：bargain 分布；语言 level × bargain（箱线或柱状）
- `结论.md`

依赖：需要议题2的语言列；在 `main` 里传入已带语言的表即可。

---

## 7. 入口：`main.py`

```text
1. 读 dataset/dataset.csv
2. df = add_eur_columns(df)
3. fair_df = fair_pay_analysis.run(df)          # → results/01_…
4. lang_df = language_scoring.score_languages(df)
5. language_scoring.analyze_vs_fair_pay(lang_df, fair_df)  # → results/02_…
6. bargain_analysis.run(lang_df)               # → results/03_…（内部再算议价）
```

运行：`python main.py`（项目根目录，已激活 venv）。
